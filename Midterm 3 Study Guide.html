Midterm 3 Study Guide

The following problems will appear on the test:

  • Define abstraction. (1-2 sentences)
  • Why is abstraction good? (1-2 sentences)
  • Write a complete program that asks the user for a file name, opens that file and reads "double" values from the file into an array (the first line of the file is an integer representing how many "double" values are in the file), closes the file, then finds the min, max, and average of the numbers.
  • Write a function called fibn that has one argument: an integer n, and returns the integer that is the "n'th" Fibonacci number. Fibonacci numbers start as follows: 0, 1, 1, 2, 3, 5, 8, ... The pattern is: each successive number is the sum of the two previous numbers. So, the ninth (n=9) Fibonacci number is 21.
  • Rewrite the following code, changing it so that is uses at least two functions besides the main function (this is the Newton example from lecture; it finds the x-value where a function hits zero, by using iterative approximation):
#include <iostream>
#include <cmath>
using namespace std;

int main()
{
        double x1, x2;

        // initial guess
        x1 = 2.0;
        x2 = x1 - (-2*sin(cos(sqrt(x1))))
                   / ((cos(cos(sqrt(x1)))*sin(sqrt(x1)))
                  / sqrt(x1));
        while(fabs(x2-x1) > 0.00001)
        {
                x1 = x2;
                x2 = x1 - (-2*sin(cos(sqrt(x1))))
                           / ((cos(cos(sqrt(x1)))*sin(sqrt(x1)))
                          / sqrt(x1));
        }
        cout << x2 << endl;
}

More hints:

  • Be able to convert set of nested if/else if/else conditionals to a non-nested equivalent version.
  • Be able to say what a piece of code "prints", especially when that code involves loops, arrays, and/or functions.
  • Know how to write a double sum or triple sum with for loops (e.g., write as nested for loops).
  • Know how to convert a for loop to a while loop.
